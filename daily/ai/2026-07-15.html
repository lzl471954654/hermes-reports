GPT-5.6 发布：三个模型，一个信号——「比 Fable 便宜，比 Fable 好用」

7月9日，OpenAI 正式发布 GPT-5.6 系列——Sol（旗舰）、Terra（性价比）、Luna（低延迟）。这不是一次普通的模型迭代，而是 OpenAI 对 Anthropic 的最直接回应。

关键数据：

  • Agent's Last Exam：Sol 拿下 53.6 分，远超 Claude Fable 5（adaptive reasoning）的 40.5 分，领先 13.1 分
  • SWE-Bench Pro：Sol 仍落后 Claude Mythos 5 和 Fable 5 约 15 分——agent coding 不是靠蛮力推理就能赢的
  • 定价革命：Sol $5/$30 vs Fable 5 $10/$50；Terra 价格是 GPT-5.5 的一半；Luna 在部分场景可用 1/16 的成本达到竞品同等效果

新 API 能力（这才是重点）：程序化工具调用（programmatic tool calling）、多智能体支持、提示缓存断点（prompt cache breakpoints）。这三项让 agent 开发者可以更高效地构建复杂工作流——不再需要 hack 式的 workaround。

💡 深度分析：GPT-5.6 的真正意义不在 benchmark 数字，而在于 OpenAI 终于不再只卷「最强模型」，开始认真做「模型家族」和「开发者体验」。Sol 在 Agent's Last Exam（一个衡量真实世界专业工作流的评测）上的大幅领先说明：OpenAI 在 agent 长程任务上找到了突破口。但 SWE-Bench Pro 的差距也暴露了——编程 agent 需要的不只是推理能力，还有对代码库的理解、工具使用习惯、以及长上下文中的精准定位。这些都是 Anthropic 的护城河。对 agent 开发者来说：如果你做的是通用 agent 工作流，Sol 的 API 性价比和程序化工具调用值得立刻切过去；如果主打 coding agent，Fable 5 仍是更成熟的选项。

Grok 4.5：用 Cursor 用户数据训练出的 coding agent

7月8日，SpaceXAI（原 xAI）与 Cursor 联合发布 Grok 4.5——这是 SpaceX 完成对 Cursor 的 $60B 收购后推出的首款模型。核心卖点：Grok 4.5 的训练数据包含了真实 Cursor 用户的开发工作流、debugger 交互、multi-file diffs 和用户修正数据。

Benchmark 速览：

  • Terminal Bench 2.1：83.3%，与 Fable max（84.3%）、GPT-5.5 xhigh（83.4%）几乎持平
  • SWE Bench Pro：64.7%，落后 Fable max（80.4%）和 Opus 4.8（69.2%），但超过了 GPT-5.5
  • DeepSWE 1.0：62.0%，位列第三（Fable 66.1%，GPT-5.5 64.31%）
  • SWE Marathon 解决率：29.0%，领先所有竞品（Opus 4.8 26.0%，Fable 24.0%）

定价：$2/$6 per million tokens，80 TPS——这个价格是 Fable 5 的 1/5。

💡 深度分析：Grok 4.5 最值得关注的是它的训练方法论——用真实开发者行为数据做 alignment，而不只是静态代码库。这让它在 SWE Marathon（长链条编码任务）上拿了第一。本质上，Grok 4.5 是在复制人类开发者「在真实环境中调试和迭代」的模式。但要注意：它的 SWE Bench Pro 仍然落后第一梯队 15 分以上，说明真实数据 + 强化学习这条路线有潜力但还没跑通。对 agent 开发者：$2/M input 的价格 + 80 TPS 的吞吐量，让 Grok 4.5 成为目前最具性价比的 coding agent 基座模型——如果你的 agent 需要大规模、高频次的代码生成和调试，这可能是最经济的选择。

Anthropic 层级 Agent Spawning：子智能体可以生孙智能体

6月10日，Claude Code v2.1.172 引入了层级 agent spawning——父 agent 可以创建子 agent，子 agent 可以再创建孙 agent，最多 5 级深度。每一级有独立的 context window，token 成本按分支累计。

同期，Anthropic 在东京发布了 MCP 社区工具市场，包含 20+ 法律垂直领域 MCP 连接器和 12 个实践领域插件。6月5-6日的发布还引入了：fallback model chains、per-agent cost attribution、scoped permissions。

💡 深度分析：五级 agent spawning 不是在炫技，而是 Anthropic 在认真解决「复杂任务如何分解」这一 agent 核心问题。但研究社区已经发现了三个关键反模式：runaway nesting（子agent无限递归）、Opus-everywhere（每级都用最强模型导致成本爆炸）、fake allowlists（权限范围被继承突破）。这意味着：层级 agent 架构强大但危险，需要开发者对每一级的资源配置和权限边界有精确控制。对 agent 开发者：如果你在做复杂的多步骤任务（代码库迁移、大规模重构、跨服务调试），层级 agent 是下一个值得实验的方向。但请务必配上 cost attribution 和 scope enforcement。

Agent Test-Time Scaling 研究突破：不是算力越多越好

Meta 与学术界合作发布论文《Scaling Test-Time Compute for Agentic Coding》（arXiv: 2604.16529），提出 agent test-time compute 的核心问题不是计算量，而是表示（representation）、选择（selection）和复用（reuse）。他们的框架使用 Recursive Tournament Voting (RTV) 做并行选择，Parallel-Distill-Refine (PDR) 做顺序推理。

同期，CMU 的研究指出：agent test-time scaling 存在天花板——对于 agentic coding 任务，简单扩展推理轮次不如「任务分解 + 定向重试」有效。

💡 深度分析：这对 agent 开发者的启示非常具体：不要盲目给 agent 加推理轮次或提高 temperature。而是应该：并行跑多个 rollouts → 结构化总结 → 投票选出最优路径 → 只在关键步骤做精细化优化。这个方法可以直接集成到现在的 agent pipeline 中，不需要等新模型。

💭 深度思考

思考一：Agent 赛道正在从「军备竞赛」转向「工程竞赛」

本周的三件大事——GPT-5.6 发布、Grok 4.5 发布、Zuckerberg 认错——指向同一个结论：模型能力不再是 agent 落地的瓶颈，工程能力才是。

GPT-5.6 Sol 的推理能力已经很强了，但它在 SWE-Bench Pro 上仍然落后 Fable 5 15 分。Grok 4.5 用真实开发者数据训练，在 SWE Marathon 上拿了第一但在 SWE Bench Pro 上只有 64.7%。Zuckerberg 砸了 $145B 却承认 agent 没达到预期。这些事实共同说明：

让一个聪明模型「会推理」和让它「可靠地完成一个 2 小时的编程任务」之间，差着十万八千里。中间需要补的，是状态管理、错误恢复、工具调用一致性、长程记忆、权限控制——这些都不是 scaling law 能解决的。

对 agent 开发者的 actionable 建议：不要等下一个更强的模型，把你的精力花在 agent 基础设施上——更好的 tool description、更鲁棒的 fallback 机制、更好的 context 管理。这些才是决定你的 agent 是「demo」还是「product」的关键。

思考二：小模型 + 垂直深耕是 agent 的暗线机会

Leanstral 1.5 用 6B 参数在定理证明上做到极致，端侧 1-3B 模型在特定场景可以匹配 GPT-4 的工具调用能力。2026 年有一个被低估的趋势：通用大模型做「大脑」+ 垂直小模型做「手和脚」的混合架构。

想象一个 coding agent：用 Grok 4.5 或 Sol 做高层任务规划和意图理解，用微调过的 3B 模型做 linting、格式化、简单重构、测试生成。前者的成本是后者的 100 倍，但 80% 的 agent 调用其实不需要那么强的模型。

Anthropic 的层级 agent spawning 架构天然适合这种混合模式——父 agent 用大模型规划，子 agent 用小模型执行，成本和效果同时优化。这个方向值得全力跟进。
