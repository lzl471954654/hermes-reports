💭 深度思考

一、2026 年 7 月：Agent 从「能不能做」切换到「多便宜能做」

本周最值得关注的不是任何一个单独的模型，而是GPT-5.6 Luna ($1/M)、Muse Spark 1.1 ($0.26/task)、Leanstral (6.5B 激活)、North Mini Code (3B 激活)这四个数字放在一起透露的信号：

2025 年的 agent 开发是在问「这个模型能不能完成这个任务」，2026 年 7 月的问题变成了「用哪个模型最划算」。这是一个范式的转变。

对 agent 开发者的 actionable insight：

  • 做 agent router：不要绑定单一模型。简单任务路由给 Luna/North Mini Code，复杂推理路由给 Sol——成本可能差 10 倍。
  • 拥抱重试：当单次 agent 任务成本低到 $0.01-$0.26，你完全可以允许 agent 失败 3-5 次并自动重试——整体成本仍然低于一次「完美 prompt」的 token 消耗。
  • 本地化加速：PyTorch 2.13 + MoE 小模型意味着本地 agent 正在从「可行」变成「实用」。考虑将敏感任务的推理留在本地。

二、Agent 安全的「Mythos 时刻」——行业正在觉醒

同一个月内，我们看到三个独立的安全信号交织在一起：

  1. Anthropic 扣留 Mythos——因为 blast radius 太大
  2. DeepMind 发布 insider-threat 模型——放弃 alignment 假设，转向系统级控制
  3. OpenAI GPT-5.6 的安全评估——明确标注在 ExploitGym 等网络安全基准上能力提升

这三条线汇聚成一个清晰的信号：agent 能力越强，遏制（containment）就越重要。但目前的 agent 框架——LangGraph、CrewAI、AutoGen——在安全设计上仍然非常原始。

对 agent 开发者的 actionable insight：

  • 在设计 agent 架构时，把每个 tool 的权限边界显式建模——不要给 agent 「sudo」权限。
  • 实现 human-in-the-loop 的「梯级授权」：读文件自动通过，写文件需要确认，执行 shell 需要审批。
  • 关注 MCP 协议的安全扩展——它很可能成为 agent 权限管理的标准层。
